Create Free Accounts
You need 3 accounts — all free — takes about 3 minutes total
🐙

1. GitHub Account

GitHub stores your code and connects to Railway and Vercel automatically.

  1. Open your browser and go to github.com
  2. Click the green "Sign up" button
  3. Enter your email address → create a password → choose a username (e.g. ali_developer)
  4. GitHub sends a verification code to your email — enter it
  5. On the welcome screen, click "Skip personalization" at the bottom
  6. ✅ You now have a GitHub account
💡 If you already have a GitHub account, skip this and move to Railway.
🚂

2. Railway Account

Railway hosts your Django backend and PostgreSQL database. Free tier includes $5/month credit — more than enough.

  1. Go to railway.app
  2. Click "Login" in the top right
  3. Click "Login with GitHub"
  4. A popup appears asking you to authorize Railway — click "Authorize"
  5. You are now logged into Railway with your GitHub account
💡 No credit card required for the free tier. Railway just needs your GitHub account.
▲

3. Vercel Account

Vercel hosts your Next.js frontend. It is completely free for this type of project with no limits.

  1. Go to vercel.com
  2. Click "Sign Up"
  3. Click "Continue with GitHub"
  4. Authorize Vercel when the popup appears
  5. On the "Tell us about yourself" screen — select "Personal" and click Continue
  6. ✅ You now have a Vercel account
✅ All 3 accounts ready. Time to upload the code.

Upload Code to GitHub
Create a repository and push your project files
📁

Create a New GitHub Repository

  1. Go to github.com and make sure you are logged in
  2. Click the "+" icon in the very top-right corner of the page
  3. Click "New repository"
  4. In the "Repository name" field, type: carmarket
  5. Under visibility, select "Public" (required for free Railway deployment)
  6. Make sure all three checkboxes are UNCHECKED — do NOT add README, .gitignore, or license (your code already has these)
  7. Click the green "Create repository" button

You now see an empty repository page. Keep this tab open.

📤

Upload Your Code Files

Choose the method that suits you:

Method A — Drag and Drop (No terminal needed, easiest):

  1. On the empty GitHub repository page, you will see a link that says "uploading an existing file" — click it
  2. Open your File Explorer / Finder on your computer
  3. Find the unzipped carmarket folder you downloaded
  4. Open the folder so you can see its contents: backend/, frontend/, README.md, etc.
  5. Select ALL files and folders inside (Ctrl+A on Windows, Cmd+A on Mac)
  6. Drag them all into the GitHub upload box in your browser
  7. Wait for all files to finish uploading (you'll see a list appear)
  8. Scroll to the bottom of the page
  9. Click the green "Commit changes" button
⚠️ Upload the contents of the carmarket folder — not the folder itself. You should see backend/, frontend/, README.md listed after upload.

Method B — Using Git Terminal (faster if you know terminal):

cd /path/to/your/carmarket
git init
git add .
git commit -m "Initial commit - CarMarket project"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/carmarket.git
git push -u origin main
⚠️ Replace YOUR_USERNAME with your actual GitHub username.
✅ Your code is now on GitHub at: github.com/YOUR_USERNAME/carmarket

Deploy Backend on Railway
Deploy the Django API — auto-detects Python, installs everything
🚀

Create New Project on Railway

  1. Go to railway.app and log in
  2. You land on your Railway dashboard
  3. Click the purple "New Project" button
  4. In the popup menu, click "Deploy from GitHub repo"
  5. If you see "Configure GitHub App" — click it, it opens a GitHub permissions page. Click "Only select repositories", choose carmarket, click "Save"
  6. Back on Railway, your carmarket repo now appears in the list — click it
📂

Set Root Directory to "backend"

This is the most important step. Railway needs to know the Django app is in the backend/ subfolder.

  1. After selecting the repo, a configuration drawer slides in from the right
  2. Look for the field labelled "Root Directory"
  3. Click on it and type exactly: backend
  4. Click "Deploy Now"
⚠️ If you don't see "Root Directory" at this screen, deploy first, then go to: Service → Settings → Source → Root Directory, set it to backend, then redeploy.
👀

Watch the Build Logs

  1. Click on your new service box in Railway
  2. Click the "Deployments" tab
  3. Click on the latest deployment
  4. Watch the build log — you will see it installing Python packages
  5. The first build takes 2–4 minutes
  6. It may show a database error at the end — that is normal, we fix it in Step 3
💡 Look for Successfully installed Django in the logs — that confirms packages installed correctly.
✅ Railway has deployed your Django service. Database error is expected — fix in next step.
3
Add PostgreSQL Database
One-click database — Railway connects it automatically
🐘

Add PostgreSQL to Your Railway Project

  1. In Railway, make sure you are in your project view (you can see your Django service as a box)
  2. Click the "+ New" button — it appears at the top of the project view or as a floating button
  3. Click "Database"
  4. Click "Add PostgreSQL"
  5. Railway creates a PostgreSQL database in about 30 seconds
  6. You will now see TWO boxes in your project: your Django service and a PostgreSQL box
💡 Railway automatically creates a DATABASE_URL environment variable and shares it with your Django service. The Django settings are already configured to read this variable.
🔄

Trigger a Redeploy

  1. Click on your Django service box
  2. Go to the "Deployments" tab
  3. Click the three-dot menu (...) on the latest deployment
  4. Click "Redeploy"
  5. Wait 2–3 minutes for it to complete

This time the deploy should succeed fully. The Procfile runs python manage.py migrate automatically — this creates all your database tables.

✅ Database is connected. Django migrations ran. Your API is now working.
4
Set Environment Variables on Railway
Configure secrets and get your backend URL
🔑

Add Required Variables

  1. Click your Django service box in Railway
  2. Click the "Variables" tab in the top navigation
  3. Click "New Variable" to add each variable below one by one
Variable NameValueHow to get it
SECRET_KEYREQUIRED A long random string Go to djecrety.ir → click Generate → copy the result and paste it here
DEBUG False Type exactly: False
ALLOWED_HOSTS .railway.app,.up.railway.app Copy-paste this value exactly
CORS_ALLOW_ALL True Type exactly: True — allows your Vercel frontend to connect
⚠️ Do NOT add DATABASE_URL manually — Railway already set it automatically when you added PostgreSQL.
🌐

Generate Your Public Backend URL

  1. Click your Django service → click "Settings" tab
  2. Scroll down to the "Networking" or "Domains" section
  3. Click "Generate Domain"
  4. Railway gives you a URL like:
https://carmarket-production-a1b2c3.up.railway.app
  1. Copy this URL and save it — you will need it in Step 5
  2. Test your API by visiting this URL in your browser:
https://YOUR-RAILWAY-URL.up.railway.app/api/cars/categories/

You should see a JSON response like {"count":8,"results":[{"id":1,"name":"Economy"...}

If you see JSON — your backend is working perfectly. ✅

✅ Backend is live. You have your Railway URL. Ready for frontend deployment.
5
Deploy Frontend on Vercel
Deploy the Next.js website — connects automatically to your Railway backend
▲

Import Your Project on Vercel

  1. Go to vercel.com and log in
  2. From your dashboard, click "Add New…" then click "Project"
  3. Under "Import Git Repository", you should see your carmarket repo
  4. If you don't see it, click "Adjust GitHub App Permissions" and grant access to the carmarket repo
  5. Click "Import" next to carmarket
📂

Set Root Directory to "frontend"

Just like Railway needed backend, Vercel needs to know the Next.js app is in frontend/.

  1. On the configuration screen, find the "Root Directory" field
  2. Click the "Edit" button next to it
  3. A file browser appears — click on the "frontend" folder
  4. Click "Continue"
⚠️ This step is critical. Without it, Vercel builds the wrong directory and the deployment fails.
🔑

Add the Environment Variable

  1. Still on the Vercel configuration screen, scroll down to "Environment Variables"
  2. In the Name field, type: NEXT_PUBLIC_API_URL
  3. In the Value field, paste your Railway URL and add /api at the end:
https://carmarket-production-a1b2c3.up.railway.app/api
⚠️ Make sure it ends with /api — not just the domain. Without this, no cars will load on the website.
🚀

Click Deploy

  1. Click the "Deploy" button
  2. Vercel builds your Next.js app — takes about 2 minutes
  3. Watch the build progress bar
  4. When done, you see a success screen with your live URL
https://carmarket-yourusername.vercel.app

Click "Continue to Dashboard" to see your project.

🔄

Update Railway with Frontend URL

This step lets the Django backend accept requests from your Vercel frontend.

  1. Copy your full Vercel URL (e.g. https://carmarket-yourusername.vercel.app)
  2. Go back to Railway → your Django service → Variables tab
  3. Click "New Variable"
  4. Name: FRONTEND_URL
  5. Value: your Vercel URL (e.g. https://carmarket-yourusername.vercel.app)
  6. Click "Add" — Railway automatically redeploys
✅ Frontend is live on Vercel! Both services are connected. The website is online.

Troubleshooting
Common errors and exact fixes

❌ Railway build fails with "No module named..." or "Module not found"

Fix: The Root Directory is not set correctly.

  1. In Railway, click your Django service → Settings tab
  2. Find "Source" section → "Root Directory"
  3. Make sure it says backend (not empty, not /backend)
  4. Click Save → Redeploy

❌ Website shows "Network Error" or no cars load

Fix: The NEXT_PUBLIC_API_URL on Vercel is wrong.

  1. Go to Vercel → your project → Settings → Environment Variables
  2. Check the value of NEXT_PUBLIC_API_URL
  3. It must be: https://YOUR-RAILWAY-URL.up.railway.app/api (with /api at the end, using https://)
  4. Edit the variable → Save → Go to Deployments → Redeploy

❌ Login returns "No active account found" or "Invalid credentials"

Fix: Seed data hasn't been run yet.

  1. Open Railway shell and run: python manage.py seed_data
  2. If already run, try: python manage.py shell -c "from django.contrib.auth import get_user_model; print(get_user_model().objects.count())"
  3. If count shows 0, run seed_data again

❌ CORS error in browser developer console

Fix: Set CORS to allow all origins.

  1. Railway → Django service → Variables tab
  2. Add/update: CORS_ALLOW_ALL = True
  3. Railway auto-redeploys. Wait 2 minutes and try again.

❌ Vercel build fails with "Module not found"

Fix: Root Directory for Vercel is not set to frontend.

  1. Vercel → your project → Settings → General
  2. Find "Root Directory" — it should say frontend
  3. If it's blank or wrong, click Edit → type frontend → Save
  4. Go to Deployments → Redeploy

❌ Images are not showing (broken image icons)

This is expected on Railway's free tier. Railway does not have persistent file storage — uploaded images don't survive between deploys. The 25 seeded cars use placeholder images which always work.

For production with real images, you would need to add Cloudinary (free tier available). This is a future enhancement beyond the current scope.

❌ Railway deployment keeps restarting / crashing

Fix: Missing SECRET_KEY environment variable.

  1. Railway → Django service → Variables tab
  2. Make sure SECRET_KEY is set to a long random string
  3. Go to djecrety.ir, generate a key, paste it as the value
  4. Railway redeploys automatically

❌ "OperationalError" or "relation does not exist" in Railway logs

Fix: Migrations haven't run yet.

  1. Open Railway shell and run:
python manage.py migrate --run-syncdb

Then run python manage.py seed_data again.
